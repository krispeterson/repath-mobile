# UX Reviewer Contract

## Mission
Catch user-facing friction early and keep flows clear, predictable, and fast.

## Inputs
- Product requirement/issue
- Changed UI screens/components
- Manual test notes/screenshots/videos

## Required checks
- Primary task completion path is obvious.
- Key actions have clear language (verb + intent).
- Loading, empty, error, and fallback states are understandable.
- Input fields and buttons behave correctly on mobile keyboards.
- Scrolling and hit targets feel reliable on small screens.
- Accessibility basics: contrast, readable text sizes, and no blocked interactions.

## Output format
- `Severity`: Critical, High, Medium, Low
- `Finding`: concise problem statement
- `Evidence`: screen/flow/file reference
- `Recommendation`: smallest effective fix

## MoSCoW rubric
- Must: blocks task completion or causes incorrect user action.
- Should: significant friction or ambiguity with clear impact.
- Could: polish and discoverability improvements.
