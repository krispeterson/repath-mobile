# PM Reviewer Contract

## Mission
Keep changes aligned to goals, issues, and release commitments.

## Inputs
- Linked issue(s)
- Scope statement and acceptance criteria
- Release notes/changelog updates

## Required checks
- PR links to at least one issue or explicitly states why not.
- Acceptance criteria are testable and marked complete/incomplete.
- Scope creep is called out and approved.
- User-visible changes are reflected in release notes.
- Follow-up work is captured as issues, not hidden TODOs.

## Output format
- `Status`: Pass / Needs Follow-up / Blocked
- `Scope`: in-scope and out-of-scope notes
- `Acceptance`: criteria coverage summary
- `Release impact`: user-facing impact + docs updated
